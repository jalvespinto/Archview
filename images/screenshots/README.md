## Screenshot Guidelines

### Technical Requirements
- **Format**: PNG with transparency where applicable
- **Resolution**: High DPI (2x or 3x for retina displays)
- **Compression**: Optimize for web (use tools like TinyPNG)
- **File size**: Keep under 500KB per image

### Content Guidelines
- Use a clean, professional sample project
- Ensure code is readable and well-formatted
- Use dark theme for consistency
- Avoid sensitive or proprietary code
- Include realistic but simple examples

### Visual Guidelines
- Ensure UI elements are clearly visible
- Use arrows or highlights to draw attention to key features
- Maintain consistent styling across screenshots
- Ensure text is legible at all sizes

## Creating Screenshots

### Tools
- **macOS**: Cmd+Shift+4 (selection), Cmd+Shift+3 (full screen)
- **Windows**: Snipping Tool, Win+Shift+S
- **Linux**: gnome-screenshot, Spectacle, Flameshot

### Editing
- Use tools like Photoshop, GIMP, or Figma
- Add annotations if needed (arrows, highlights)
- Crop to remove unnecessary UI
- Ensure consistent dimensions

### Optimization
- Use ImageOptim (macOS), TinyPNG (web), or similar
- Target 200-500KB per image
- Maintain visual quality
